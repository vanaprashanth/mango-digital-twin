# Daily Pipeline Refresh — Windows Setup Guide

This guide explains how to run the Mango Digital Twin pipeline automatically every day on Windows using Task Scheduler.

---

## Background: two pipeline modes

| Command | What it does |
|---|---|
| `python main.py` | **Full refresh** — fetches fresh raw data from NASA POWER, Open-Meteo, and SoilGrids, then regenerates all downstream processed outputs. Use this for the daily automated run. |
| `python main.py --skip-fetch` | **Offline / rebuild** — skips all network calls and recomputes outputs from the raw data already on disk. Use this for local testing when you do not want to re-fetch. |

The dashboard (`streamlit run app/streamlit_app.py`) reads the latest processed CSV files. It does not fetch data itself; it shows whatever was last generated by the pipeline.

---

## Step 1 — Manually test the script

Open PowerShell in the project root and run:

```powershell
cd E:\mango-digital-twin
.\scripts\run_daily_pipeline.ps1
```

You should see timestamped log output in the terminal and a new log file created under `logs\daily_pipeline\`, for example:

```
logs\daily_pipeline\daily_pipeline_2025-06-15_06-00-01.log
```

Confirm:
- The script prints `SUCCESS: Pipeline completed successfully`.
- `data/processed/pipeline_run_metadata.json` has an updated `run_completed_at` timestamp.
- One or more processed CSV files were updated (check `git status`).

If you see `FAILURE`, read the log file for details — see the Troubleshooting section below.

> **Note:** If PowerShell blocks script execution, run once as Administrator:
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
> ```
> Or run the script directly with `-ExecutionPolicy Bypass` as shown in the Task Scheduler setup below.

## Step 2 — Create a Windows Task Scheduler task

### Option A — Task Scheduler GUI

1. Open **Task Scheduler** (search in Start menu).
2. Click **Create Task** (not "Create Basic Task" — you need the full editor).

**General tab**
- Name: `MangoDailyPipeline`
- Description: `Daily full refresh of the Mango Digital Twin pipeline`
- Run whether user is logged on or not: ✓ (check this)
- Run with highest privileges: ✓ (check this)

**Triggers tab**
- Click **New**
- Begin the task: `On a schedule`
- Settings: `Daily`
- Start: `06:00:00` (6:00 AM local time)
- Recur every: `1` days
- Enabled: ✓

**Actions tab**
- Click **New**
- Action: `Start a program`
- Program/script: `powershell.exe`
- Arguments:
  ```
  -ExecutionPolicy Bypass -File "E:\mango-digital-twin\scripts\run_daily_pipeline.ps1"
  ```
- Start in: `E:\mango-digital-twin`

**Conditions tab**
- Start only if the computer is on AC power: uncheck this if you want it to run on battery too.
- Wake the computer to run this task: optional — only if the PC may be asleep at 6 AM.

**Settings tab**
- If the task is already running: `Stop the existing instance` (prevents overlapping runs).
- Stop the task if it runs longer than: `2 hours` (safety limit).

Click **OK** and enter your Windows password when prompted.

### Option B — PowerShell one-liner (run as Administrator)

```powershell
$Action  = New-ScheduledTaskAction `
    -Execute "powershell.exe" `
    -Argument '-ExecutionPolicy Bypass -File "E:\mango-digital-twin\scripts\run_daily_pipeline.ps1"' `
    -WorkingDirectory "E:\mango-digital-twin"

$Trigger = New-ScheduledTaskTrigger -Daily -At 06:00

$Settings = New-ScheduledTaskSettingsSet `
    -ExecutionTimeLimit (New-TimeSpan -Hours 2) `
    -MultipleInstances IgnoreNew

Register-ScheduledTask `
    -TaskName "MangoDailyPipeline" `
    -Action $Action `
    -Trigger $Trigger `
    -Settings $Settings `
    -RunLevel Highest `
    -Force
```

## Step 3 — Verify the scheduled task ran

After the first scheduled run (or after triggering it manually from Task Scheduler):

1. **Check the log directory:**
   ```powershell
   ls E:\mango-digital-twin\logs\daily_pipeline\ | Sort-Object LastWriteTime -Descending | Select-Object -First 5
   ```

2. **Read the latest log:**
   ```powershell
   Get-Content (ls E:\mango-digital-twin\logs\daily_pipeline\ | Sort-Object LastWriteTime -Descending | Select-Object -First 1).FullName
   ```

3. **Confirm metadata was updated:**
   ```powershell
   Get-Content E:\mango-digital-twin\data\processed\pipeline_run_metadata.json | Select-String "run_completed_at"
   ```

4. **Confirm the dashboard sees fresh data:** launch the dashboard and check the **Data Freshness** section in the sidebar — it shows the last-updated timestamp for each output file.

## Step 4 — Check the dashboard

```powershell
cd E:\mango-digital-twin
.\.venv\Scripts\Activate.ps1
streamlit run app/streamlit_app.py
```

The sidebar **Data Freshness** section shows the latest-date and last-modified timestamp for each processed output. If the pipeline ran successfully, these will reflect today's date.

## Log rotation (optional)

Logs accumulate over time. To delete logs older than 30 days, add this to the bottom of `run_daily_pipeline.ps1` or run manually:

```powershell
Get-ChildItem "E:\mango-digital-twin\logs\daily_pipeline\" -Filter "*.log" |
    Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) } |
    Remove-Item -Force
```

## Troubleshooting

### Internet unavailable at scheduled run time

The pipeline will fail and log an error from the fetch step. The previously generated processed files remain unchanged on disk — the dashboard will still work, just with stale data. Re-run manually once the internet is available:

```powershell
.\scripts\run_daily_pipeline.ps1
```

### API fetch failed (NASA POWER / Open-Meteo / SoilGrids)

Look for the specific error in the log file. Common causes: API rate limit, temporary outage, or the API endpoint URL changed. Wait and retry.

### Virtual environment not found

The script will print:

```
ERROR: Virtual environment not found at: E:\mango-digital-twin\.venv\Scripts\Activate.ps1
```

Recreate it:

```powershell
cd E:\mango-digital-twin
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### Package missing (ImportError or ModuleNotFoundError)

Activate the venv and reinstall:

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### Google Earth Engine authentication expired

The Sentinel-2 fetch step requires an authenticated GEE session. If it fails with an authentication error:

```powershell
.\.venv\Scripts\Activate.ps1
earthengine authenticate
```

Then re-run the pipeline.

### Data files regenerated but not committed

The pipeline writes new processed CSV files on each run. These are not committed automatically. The dashboard reads them directly from disk. If you want to version the outputs, commit them manually after reviewing:

```powershell
git add data/processed/
git commit -m "Update processed outputs YYYY-MM-DD"
git push
```

Do **not** use `git add .` — this would also stage unintended files. Stage data files explicitly.

### Task Scheduler shows "Last Run Result: 0x1"

Exit code 1 means the pipeline script itself exited with failure. Check the most recent log file in `logs\daily_pipeline\` for the specific error.

### Task ran but log file is empty or missing

Verify the **Start in** directory in the Task Scheduler action is set to `E:\mango-digital-twin`. If it is blank or wrong, the script cannot find relative paths.
